# 图文编辑器：上传图片不再要求标题

## 问题

在图文编辑器 (`ArticleEditor.tsx`) 中，上传图片时 `handleImageUpload` 会先调用 `doSave()` 创建文章。而 `doSave()` 要求标题不为空（`ArticleEditor.tsx:134`），导致用户必须先输入标题才能上传图片。

## 方案

使用媒体库 API (`POST /api/media/upload`) 上传图片（无需标题/文章ID），图片先存到 `uploads/` 目录。当文章保存时（`doSave` 成功后），再把图片拷贝到文章目录并更新 markdown 中的路径为相对路径。

### 具体改动

#### 1. 前端 - `desktop/src/pages/ArticleEditor.tsx`

**修改 `handleImageUpload`（行 183-198）：**
- 不再调用 `doSave()` 来创建文章
- 改用 `apiService.uploadFile(file)` 上传到媒体库（不依赖文章ID）
- 记录上传的媒体信息（media id、file_path）到临时列表 state（如 `pendingImages`）
- 插入 markdown 时先使用绝对路径占位符（如 `![name](/uploads/xxx.png)`）

**新增 `doSave` 后处理逻辑：**
- 在 `doSave()` 成功后（新创建文章时），检查 `pendingImages` 列表
- 调用新的后端 API 将图片从媒体库拷贝到文章目录
- 更新 content 中对应的图片路径为相对路径

#### 2. 前端 - `desktop/src/services/api.ts`

**新增 API 方法：**
```typescript
async copyMediaToArticle(articleId: number, mediaId: number): Promise<Media>
```
调用 `POST /api/media/copy-to-article/{article_id}/{media_id}`

#### 3. 后端 - `server/app/routes/media.py`

**新增路由 `POST /api/media/copy-to-article/{article_id}/{media_id}`：**
- 查找 media 记录和 article 记录
- 从 `uploads/` 目录拷贝文件到文章的 `images/` 子目录
- 更新 media 记录的 `file_path` 为相对路径，设置 `article_id`
- 返回更新后的 media 信息

### 流程对比

**之前：**
1. 用户输入标题 → 2. 上传图片（自动创建文章） → 3. 继续编辑

**之后：**
1. 用户直接上传图片（存到媒体库）→ 2. 继续编辑 → 3. 保存时（输入标题后），自动将图片拷贝到文章目录并更新路径

### 需要注意

- 编辑已有文章时（`articleId` 已存在），可以直接使用 `uploadToArticle`，无需改变
- `pendingImages` state 需要在文章创建成功后清空
- 拷贝操作是复制（非移动），原媒体库文件仍保留，避免其他引用丢失
