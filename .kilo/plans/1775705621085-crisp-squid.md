# Plan: WeChat Official Account Integration Enhancement

## Overview

Based on the WeChat Official Account API documentation, enhance the existing `wechat_mp` plugin with four features: access token management with config validation, frontend access token editing, multi-platform publishing from article list, and removal of the publish button from the article editor.

---

## Feature 1: Access Token Management via `/cgi-bin/token`

**Goal**: When binding or testing a wechat_mp platform, call `/cgi-bin/token` with `app_id`/`app_secret` to verify the configuration. On success, save the returned `access_token` and `expires_in` to the `platform_accounts` database record. Frontend can also edit the access_token.

### Backend Changes

#### 1.1 Update `WeChatMPPlugin.test_connection()` (`server/app/plugins/wechat_mp.py`)
- Change signature to accept optional `db` session parameter
- After successfully calling `/cgi-bin/token`, update the `PlatformAccount` record:
  - Set `access_token` to the returned token
  - Set `token_expires_at` to `now + expires_in seconds`
  - Set `status` to `"active"`
- Return a dict with `{ connected: bool, status: str, access_token_saved: bool }` instead of just `bool`

#### 1.2 Update `POST /api/platforms/{account_id}/test` route (`server/app/routes/platforms.py`)
- Pass `db` session to `plugin.test_connection(account, db)`
- Update account status based on result
- Return the full result dict including `access_token_saved`

#### 1.3 Add access_token to `PlatformAccountResponse` schema (`server/app/schemas/platform.py`)
- Add `access_token: Optional[str] = None` to `PlatformAccountResponse`
- This allows the frontend to display/edit the saved access_token

### Frontend Changes

#### 1.4 Update `PlatformManager.tsx` config modal
- In the "微信公众号配置" modal, add an `access_token` field (read-only or editable `Input`)
- Show the current access_token and its expiration status
- After config save + auto-test succeeds, refresh the displayed access_token

## Feature 2: Frontend-Provided Access Token Validation via `/cgi-bin/get_api_domain_ip`

**Goal**: When the frontend provides an `access_token` during platform configuration, validate it by calling `/cgi-bin/get_api_domain_ip`. If it fails, fall back to getting a new access_token via `/cgi-bin/token` and validate that instead.

### Backend Changes

#### 2.1 Add `_validate_access_token()` method to `WeChatMPPlugin` (`server/app/plugins/wechat_mp.py`)
- Call `GET /cgi-bin/get_api_domain_ip?access_token=TOKEN`
- If returns `ip_list` -> token is valid, return `True`
- If returns error (errcode != 0) -> token is invalid, return `False`

#### 2.2 Add `_get_or_refresh_access_token()` method
- If `account.access_token` exists and is not expired, validate it via `get_api_domain_ip`
- If valid, return it
- If invalid or missing, call `/cgi-bin/token` to get a new one
- Update the database record with new token and expiry

#### 2.3 Add a new endpoint `POST /api/platforms/{account_id}/validate-token` (`server/app/routes/platforms.py`)
- Accept optional `access_token` in request body
- If provided, validate via `get_api_domain_ip`
- If valid, save to DB and return success
- If invalid, try `/cgi-bin/token` to get a fresh token, validate it, save to DB
- Return result with `valid`, `access_token` (if refreshed), `status`

#### 2.4 Update `test_connection()` to support token validation flow
- If `account.access_token` exists, first try validating it
- If validation fails, try getting a fresh token via app_id/app_secret
- Update DB accordingly

### Frontend Changes

#### 2.5 Add access_token field to bind form in `PlatformManager.tsx`
- When binding wechat_mp, show an optional "Access Token" field
- If user provides an access_token, send it along with the config
- Backend will validate it

#### 2.6 Update config modal to show access_token with validation
- Show current access_token in the config modal
- Add "Validate Token" button that calls the validate endpoint
- Show validation result (valid/invalid/refreshed)

## Feature 3: Multi-Platform Publishing from Article List

**Goal**: In the article list (`ArticleList.tsx`), add a "Publish" button to each article row. Clicking it opens `PublishModal` to select active platforms. For wechat_mp, the publish flow must: (a) upload images via `/cgi-bin/media/uploadimg`, (b) create draft via `/cgi-bin/draft/add`, (c) submit for publishing via `/cgi-bin/freepublish/submit`.

### Backend Changes

#### 3.1 Fix image upload in `WeChatMPPlugin.publish()` (`server/app/plugins/wechat_mp.py`)
- **Current issue**: Only uploads the first image as thumb via `add_material`, and does not upload content images to WeChat CDN
- **Fix**: Parse all `![alt](path)` in article content
  - For each local image, call `/cgi-bin/media/uploadimg` to upload to WeChat CDN
  - Replace the local image path with the returned WeChat URL in the content
- This ensures article content has valid WeChat CDN URLs that pass their validation

#### 3.2 Keep the existing draft + publish flow
- `_upload_thumb_media()` → upload cover image as permanent material → get `thumb_media_id`
- `_add_draft()` → create draft with content (now with CDN URLs) → get `media_id`
- `_submit_publish()` → submit draft for publishing → get `publish_id`
- Add detailed `PublishLog` entries at each step

#### 3.3 Use cached/refreshed access token in `publish()`
- Replace `_get_access_token()` call with `_get_or_refresh_access_token()` which uses the DB-stored token when valid

### Frontend Changes

#### 3.4 Add "Publish" button to `ArticleList.tsx` table
- Import `PublishModal` component
- Add a `publishModalOpen` state and `publishingArticleId` state
- In the actions column, add a "发布" button (with `SendOutlined` icon) alongside "编辑" and "删除"
- Clicking it sets `publishingArticleId` and opens `PublishModal`
- On publish success, refresh the article list and publish summary

## WeChat API Reference

1. **Get Access Token**: `GET /cgi-bin/token?grant_type=client_credential&appid=APPID&secret=APPSECRET` → `{ access_token, expires_in }`
2. **Get API Domain IPs**: `GET /cgi-bin/get_api_domain_ip?access_token=TOKEN` → `{ ip_list: [...] }` (validates token)
3. **Upload Content Image**: `POST /cgi-bin/media/uploadimg?access_token=TOKEN` (form-data: `media=@file`) → `{ url }` — returns CDN URL for use in article content
4. **Add Draft**: `POST /cgi-bin/draft/add?access_token=TOKEN` body: `{ articles: [{ title, content, thumb_media_id, ... }] }` → `{ media_id }`
5. **Submit Publish**: `POST /cgi-bin/freepublish/submit?access_token=TOKEN` body: `{ media_id }` → `{ publish_id, publish_id }`
