# 文章预览窗口：将"公众号/通用"替换为"手机端/桌面端"

## 目标
将文章编辑器右侧预览窗口的平台切换从"公众号/通用"改为"手机端/桌面端"，手机端以 680×1024（宽×高）竖屏比例展示。

## 变更清单

### 1. `desktop/src/pages/ArticleEditor.tsx` (第332-333行)
- `<Radio.Button value="wechat_mp">公众号</Radio.Button>` → `<Radio.Button value="mobile">手机端</Radio.Button>`
- `<Radio.Button value="common">通用</Radio.Button>` → `<Radio.Button value="desktop">桌面端</Radio.Button>`

### 2. `desktop/src/components/Preview/PlatformPreview.tsx`
- `PlatformKey` 类型：`'common' | 'wechat_mp'` → `'mobile' | 'desktop'`
- `platformClassMap`：两个平台都映射到 `'preview-common'`
- `label`：`mobile` → `'手机端预览'`，`desktop` → `'桌面端预览'`
- **`mobile`**：使用 PhoneFrame 包裹（带手机外壳样式），按 680:1024 竖屏比例
- **`desktop`**：不使用 PhoneFrame，直接渲染内容区域，宽度占满容器
- 去掉 `import PhoneFrame` 不在 desktop 分支使用的判断（条件渲染）

### 3. `desktop/src/components/Preview/PhoneFrame.tsx`
- 移除固定 `width: 375`，改为按容器高度自适应
- 使用 `aspect-ratio: 1440 / 3200`（匹配手机分辨率 1440×3200 竖屏比例，即 9:20）
- 保持 `max-width: 100%`，`flex: 1` 让高度撑满
- 内容区域按新比例调整 padding 等

### 4. CSS 样式文件
- `preview-common` 和 `preview-wechat-mp` 都保留不动
- `platformClassMap.mobile` → `'preview-common'`
- `platformClassMap.desktop` → `'preview-common'`

### 5. `desktop/src/components/PlatformIcon.tsx`
- 不需要修改，`platformNameMap` 用于其他场景（发布平台管理），预览窗口标签已在 PlatformPreview 中独立设置
