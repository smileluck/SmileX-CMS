# Fix: markSaved circular dependency error

## Problem
`doSave` (line 232) has `markSaved` in its dependency array, but `markSaved` comes from `useAutoSave` which is defined **after** `doSave` (line 280). This causes "Cannot access 'markSaved' before initialization".

Additionally, `autoSaveFn` (line 275) depends on `doSave`, and `useAutoSave` depends on `autoSaveFn`, creating a circular chain:

```
useAutoSave → autoSaveFn → doSave → markSaved → useAutoSave
```

## Fix
1. **Move `useAutoSave` before `doSave`**
2. **Use a `markSavedRef`** to hold `markSaved` — `doSave` calls `markSavedRef.current()` instead of `markSaved` directly, removing it from the dependency array
3. **Use a `doSaveRef`** for `autoSaveFn` — or simply inline a minimal save in `autoSaveFn` that avoids calling `doSave`
4. **Wire `markSavedRef` after `useAutoSave`**
