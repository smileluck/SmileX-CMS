# 标签管理功能实现计划

## 概述

当前文章的 `tags` 字段以 JSON 数组存储（自由文本），没有独立的标签管理。本计划新增独立的 **tags 表**和 **article_tags 关联表**，实现规范化标签管理，包括：标签 CRUD、文章-标签多对多关联、按标签筛选文章、标签颜色、标签统计。

## 数据库变更

### 新增表：tags
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| name | VARCHAR(50) | 标签名（用户内唯一） |
| color | VARCHAR(20) | 标签颜色（hex，如 #1890ff），可选 |
| user_id | INTEGER FK(users.id) | 所属用户 |
| created_at | DATETIME | 创建时间 |

约束：`(name, user_id)` UNIQUE。

### 新增关联表：article_tags
| 字段 | 类型 | 说明 |
|------|------|------|
| article_id | INTEGER FK(articles.id) | 文章ID |
| tag_id | INTEGER FK(tags.id) | 标签ID |

约束：联合主键 `(article_id, tag_id)`。

### 迁移 articles.tags
- 新增关联表和 tags 表后，将现有 articles 表的 `tags` JSON 字段数据迁移到 `article_tags` 表。
- 迁移完成后保留 `tags` 列（兼容），但新逻辑使用关联表。

## 后端实现

### 1. Model：`server/app/models/tag.py`
- `Tag` 模型：id, name, color, user_id, created_at
- `ArticleTag` 关联模型：article_id, tag_id
- Tag 与 Article 的多对多关系

### 2. Schema：`server/app/schemas/tag.py`
- `TagCreate`：name (必填), color (可选)
- `TagUpdate`：name (可选), color (可选)
- `TagResponse`：id, name, color, user_id, article_count, created_at
- `TagBatchResponse`：tags 列表（含统计）

### 3. Route：`server/app/routes/tags.py`
```
GET    /api/tags                    # 获取当前用户所有标签（含文章数统计）
POST   /api/tags                    # 创建标签
PUT    /api/tags/{id}               # 更新标签
DELETE /api/tags/{id}               # 删除标签（解除所有文章关联）
GET    /api/tags/{id}/articles      # 获取标签下的文章列表
```

### 4. 修改现有文件
- **`server/app/models/__init__.py`**：导入 Tag, ArticleTag
- **`server/app/models/article.py`**：添加 `tags_rel` relationship（多对多）
- **`server/app/main.py`**：注册 tags router
- **`server/app/schemas/article.py`**：ArticleResponse 中 tags 改为从关联表读取 TagResponse 列表
- **`server/app/routes/articles.py`**：
  - 创建/更新文章时同步处理 article_tags 关联
  - 文章列表支持 `?tag_id=` 筛选参数

## 前端实现

### 1. Types：`desktop/src/types/index.ts`
- 新增 `Tag`、`TagCreate`、`TagUpdate` 接口
- 更新 `Article` 接口：`tags` 改为 `Tag[]`（或保持 string[] 兼容）

### 2. API Service：`desktop/src/services/api.ts`
- `getTags()` → `Tag[]`
- `createTag(data)` → `Tag`
- `updateTag(id, data)` → `Tag`
- `deleteTag(id)` → void
- `getTagArticles(tagId)` → `Article[]`

### 3. Store：`desktop/src/store/tagSlice.ts`

- 管理 tags 数组和 loading 状态

### 4. Store 注册：`desktop/src/store/index.ts`
- 添加 tagReducer

### 5. 页面：`desktop/src/pages/TagManager.tsx`

- 删除确认
- 颜色选择器（Ant Design `ColorPicker`）

### 6. 路由：`desktop/src/App.tsx`
- 添加 `/tags` 路由 → `TagManager`

### 7. 导航：`desktop/src/components/Layout.tsx`
- 在 menuItems 中添加标签管理入口（图标 `TagsOutlined`）

### 8. 文章列表筛选：`desktop/src/pages/ArticleList.tsx`

### 9. 文章编辑器：`desktop/src/pages/ArticleEditor.tsx`
- 标签 Select 改为从已有标签列表中选择（mode="multiple"），同时支持 `mode="tags"` 新建

## 兼容性考虑

- 文章编辑器的 `tags` 字段当前使用 Ant Design `Select mode="tags"`（自由输入）。改为从数据库标签列表选择后，仍保留 `mode="tags"` 以支持快速创建新标签。
- `Article.tags` 在前端可保持 `string[]` 显示（标签名列表），后端在 ArticleResponse 中将关联的 Tag 对象映射为名称列表 + 详细信息。
- 数据迁移：新增关联表后首次启动时，遍历 articles 表中 JSON tags 字段，自动创建 Tag 记录并建立关联。
