# 编辑器功能调整计划

## 概述
对 ArticleEditor 页面进行 4 项功能调整，涉及撤销/重做、操作历史、块级元素插入、标题标签布局。

---

## 1. 撤销/重做功能 + 操作队列管理

### 实现方案：创建 `useHistory` 自定义 Hook

**新建文件**: `desktop/src/hooks/useHistory.ts`

实现标准的 undo/redo 栈管理模式：
- 维护 `past: string[]`（历史状态栈）、`future: string[]`（重做栈）
- 暴露 `undo()`, `redo()`, `pushState(content, description)`, `clearHistory()` 方法
- `pushState` 记录快照和操作描述，栈最大容量 100
- `clearHistory()` 清空 past 和 future（手动保存时调用）
- 暴露 `canUndo`, `canRedo` 状态供按钮禁用判断
- 暴露 `history` 数组供操作历史面板显示

**修改文件**: `desktop/src/pages/ArticleEditor.tsx`
- 引入 `useHistory` hook
- 在 `setContent` 调用处，包装一层逻辑：当内容变化时，调用 `pushState` 记录变更
- 在 `doSave` 成功后调用 `clearHistory()`
- 传递 `undo`, `redo`, `canUndo`, `canRedo`, `history` 给 `EditorToolbar`

**修改文件**: `desktop/src/components/Editor/EditorToolbar.tsx`
- Props 增加 `onUndo`, `onRedo`, `canUndo`, `canRedo`
- 撤销/重做按钮绑定 onClick 和 disabled 状态

### useHistory 核心数据结构
```ts
interface HistoryEntry {
  content: string;        // 内容快照
  description: string;    // 操作描述（如 "输入文本", "插入代码块"）
  timestamp: number;      // 时间戳
}

interface UseHistoryReturn {
  pushState: (content: string, description: string) => void;
  undo: () => string | null;      // 返回恢复的内容，null 表示无法 undo
  redo: () => string | null;      // 返回恢复的内容，null 表示无法 redo
  clearHistory: () => void;
  canUndo: boolean;
  canRedo: boolean;
  history: HistoryEntry[];         // 完整操作历史（past + 当前）
  currentIndex: number;            // 当前在历史中的位置
}
```

## 3. 列表/代码块/引用/表格按钮先换行再插入

**修改文件**: `desktop/src/pages/ArticleEditor.tsx`

修改 `handleInsertMarkdown` 函数，增加 `block` 参数或在内部判断哪些元素需要前置换行：

方案：给 `onInsertMarkdown` 增加 `block?: boolean` 参数，当 `block=true` 时：
- 检查当前内容末尾是否已有 `\n`，如果没有则先加 `\n`
- 然后再插入内容

**修改文件**: `desktop/src/components/Editor/EditorToolbar.tsx`
- `EditorToolbarProps` 的 `onInsertMarkdown` 增加 `block?: boolean` 参数
- `ToolBtn` 组件增加 `block` prop
- 对以下按钮设置 `block={true}`：
  - 无序列表 (`UnorderedListOutlined`)
  - 有序列表 (`OrderedListOutlined`)
  - 引用 (`BlockOutlined`)
  - 代码块 (`CodeOutlined` 代码块)
  - 表格 (`TableOutlined`)

## 4. 标题和标签移入编辑区域上方

**修改文件**: `desktop/src/pages/ArticleEditor.tsx`

将标题/标签的 `<Form>` 从分割面板外部（全局层级）移到左侧编辑面板内部顶部：

当前布局：
```
[Action Bar]
[Title & Tags Form]          ← 全局行
[Editor + Preview Split]
  Left: [Toolbar] [Editor]
  Right: [Preview]
```

调整后：
```
[Action Bar]
[Editor + Preview Split]
  Left:
    [Title & Tags Inline]    ← 移入此处
    [Toolbar]
    [Editor]
  Right: [Preview]
```

- 标题和标签在同一行内显示（已是 `Form layout="inline"`，保持不变）
- 移到左侧编辑面板的 `flexDirection: 'column'` 容器内，置于 `EditorToolbar` 上方
- 调整样式：去掉 `marginBottom`，增加下边框分隔
- 标题 Input 去掉 `size="large"` 改为默认尺寸，节省纵向空间
