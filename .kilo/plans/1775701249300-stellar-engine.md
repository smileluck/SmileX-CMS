# Plan: 平台状态面板 + 自动测试 + 状态自动更新

## 概述

三个功能点：
1. 图文编辑右侧预览框上方增加面板，显示已配置的 active 平台和名称
2. 平台管理中，绑定/配置平台后自动测试连接（仅 test_connection，不发送文章）
3. 平台管理测试按钮根据结果更新状态（active / inactive）

---

## 任务 1: 后端 - 测试连接时自动更新平台状态

**修改文件:** `server/app/routes/platforms.py` (line 116-137)

在 `POST /api/platforms/{account_id}/test` 路由中：
- 调用 `plugin.test_connection(account)` 后
- `result == True` → `account.status = "active"`
- `result == False` → `account.status = "inactive"`
- `db.commit()` 并返回 `{ "connected": result, "status": account.status }`

---

## 任务 2: 前端 API - 新增 autoTestPlatform 方法

**修改文件:** `desktop/src/services/api.ts`

新增方法：
```ts
async autoTestPlatform(id: number): Promise<{ connected: boolean; status: string }> {
  const { data } = await this.client.post<{ connected: boolean; status: string }>(`/platforms/${id}/test`);
  return data;
}
```
（复用已有的 test 端点，后端会自动更新状态）

## 任务 4: 前端 - 绑定/配置后自动测试

**修改文件:** `desktop/src/pages/PlatformManager.tsx`

## 任务 5: 前端 - 图文编辑右侧预览框上方添加平台面板

**修改文件:** `desktop/src/pages/ArticleEditor.tsx`

在右侧预览区域的控制栏和预览内容之间插入平台面板：

### 5a. 获取平台数据
- 从 Redux store 获取平台账号列表：`useSelector((state: RootState) => state.platform.accounts)`
- 在组件 `useEffect` 中 `dispatch(fetchPlatforms())` 加载平台数据
- 筛选 `status === 'active'` 的平台

### 5b. 面板 UI
位置：在 line 528（预览控制栏 `</div>`）和 line 529（预览内容 `<div>`）之间。

```
┌──────────────────────────────────────────┐
│ [手机端] [桌面端]       同步滚动 [开关]    │  ← 已有
├──────────────────────────────────────────┤
│ 已配置平台:  🟢微信公众号:MyAccount  🟢Bilibili:Test │  ← 新增
├──────────────────────────────────────────┤
│              (预览内容区域)                │
└──────────────────────────────────────────┘
```

使用 Ant Design `Tag` 组件，每个平台显示为紧凑的 Tag（图标 + 平台名: 账号名），绿色表示 active。无 active 平台时显示"暂无已配置的平台"。
