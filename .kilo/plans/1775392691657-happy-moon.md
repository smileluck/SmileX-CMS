# SmileX-CAS 功能增强计划

## 概览

本计划涵盖 3 大功能模块的改进，涉及前端（React/TypeScript/Electron）和后端（FastAPI/Python）的改动。

---

## 功能 1：设置中增加菜单位置选项（侧边栏/顶部栏）

### 目标
在设置页面增加"菜单位置"选项，支持将导航菜单放在左侧边栏（默认，当前行为）或顶部水平栏显示。

### 前端改动

**1. 设置持久化 — `desktop/src/services/api.ts`**
- 无需后端，使用 `localStorage` 存储设置（与现有 `apiUrl` 一致）
- 新增 key `menuLayout`，值为 `'sidebar'`（默认）或 `'topbar'`

**2. Settings 页面 — `desktop/src/pages/Settings.tsx`**
- 在现有"后端 API 地址"下方增加"菜单位置"选项
- 使用 Ant Design `Radio.Group` + `Radio.Button`，选项：`sidebar`（侧边栏）、`topbar`（顶部栏）
- 保存时将选择写入 `localStorage`

**3. Layout 组件重构 — `desktop/src/components/Layout.tsx`**
- 读取 `localStorage` 中的 `menuLayout` 配置
- **侧边栏模式**（默认）：保持当前行为不变，`Sider` + `Header` + `Content`
- **顶部栏模式**：
  - 移除左侧 `Sider`
  - `Header` 中左侧放置 Logo + 水平 `Menu`（`mode="horizontal"`）
  - 右侧保留用户头像下拉
  - `Content` 占满宽度
- 添加 `useEffect` 监听 `storage` 事件，使设置变更后实时生效（或通过 React state/context）

### 不涉及后端改动

## 功能 2：图文文章编辑体验优化

### 2.1 左右分栏编辑器（Markdown 编辑 + 实时预览）

**修改文件：`desktop/src/pages/ArticleEditor.tsx`**

将现有 `<textarea>` 替换为左右分栏布局：
- **左栏**（50%）：Markdown 编辑区域，使用现有依赖中的 Milkdown 或保留 textarea + 语法高亮
- **右栏**（50%）：实时预览区域，使用已安装的 `remark-parse` → `remark-gfm` → `remark-rehype` → `rehype-stringify` pipeline 渲染 HTML
- 新建工具函数 `desktop/src/utils/markdown.ts`：封装 unified pipeline，将 markdown 文本转 HTML

**选定方案**：左侧使用已安装的 Milkdown v7 WYSIWYG 编辑器，右侧使用 unified pipeline（remark-parse → remark-gfm → remark-rehype → rehype-stringify）实时渲染预览。Milkdown 和右侧预览通过共享 Markdown 文本内容实现同步。

### 2.2 图片上传支持

**前端改动 — `desktop/src/pages/ArticleEditor.tsx`**
- 在编辑器工具栏增加"插入图片"按钮
- 点击后调用 `apiService.uploadToArticle(articleId, file)` 上传图片
- 上传成功后在 Markdown 中插入 `![filename](./{snowid}.ext)` 格式的图片引用
- 支持拖拽上传和粘贴图片

**后端改动 — `desktop/src/services/api.ts`**
- 已有 `uploadToArticle` 方法，无需改动

**后端 — `server/app/routes/media.py`**
- 已有 `POST /api/media/upload-to-article/{article_id}` 端点，无需改动
- 新建文章时需要先保存一次获取 article_id，之后才能上传图片到该文章

**预览中的图片路径处理**
- `desktop/src/utils/markdown.ts` 中处理相对路径图片，将其转换为完整的 API URL
- 使用 `apiService.getMediaUrl()` 将 `./{filename}` 转为可访问的 URL

### 2.3 保存/发布成功页面

**新建文件：`desktop/src/pages/ArticleSuccess.tsx`**
- 成功页面组件，接收参数：`articleId`, `articleTitle`, `action`（`save` | `publish`）
- 显示成功提示图标和文案
- 两个操作按钮：
  - "返回列表" → 跳转 `/articles`
  - "继续创建" → 跳转 `/articles/new`

**修改路由 — `desktop/src/App.tsx`**
- 新增路由 `/articles/success`，渲染 `ArticleSuccess` 组件
- 通过 `location.state` 传递参数

**修改 `desktop/src/pages/ArticleEditor.tsx`**
- 保存成功后：`navigate('/articles/success', { state: { articleId, articleTitle, action: 'save' } })`
- 发布成功后：`navigate('/articles/success', { state: { articleId, articleTitle, action: 'publish' } })`

### 2.4 发布功能 — 选择平台和账号

**新建组件：`desktop/src/components/PublishModal.tsx`**
- Modal 弹窗，用于选择要发布的平台和账号
- 左侧：已绑定的平台账号列表，按平台分组显示（带平台 icon）
- 每个账号前有 Checkbox 勾选
- 右侧：已选择的账号摘要
- 底部：确认发布按钮
- 数据来源：`apiService.getPlatformAccounts()` 获取已绑定账号

**修改 `desktop/src/pages/ArticleEditor.tsx`**
- "发布"按钮改为打开 `PublishModal`
- `PublishModal` 确认后调用 `apiService.createPublishTask()` 创建发布任务
- 发布任务创建成功后跳转到成功页面

### 2.5 文章列表增加平台发布状态列

**修改文件：`desktop/src/pages/ArticleList.tsx`**

**后端新增端点 — `server/app/routes/articles.py`**
- 新增 `GET /api/articles/{article_id}/publish-status` 端点
- 返回该文章在各个平台的发布状态列表：
  ```json
  [
    {
      "platform_name": "wechat_mp",
      "account_name": "我的公众号",
      "status": "success",
      "platform_post_url": "https://mp.weixin.qq.com/xxx",
      "error_message": null
    }
  ]
  ```
- 或在 `GET /api/articles` 返回时附加 `publish_status` 字段

**后端新增端点 — `server/app/routes/articles.py`**
- 新增 `GET /api/articles/publish-summary` 批量查询文章发布状态
- 根据发布任务表聚合数据

**前端 `ArticleList.tsx` 改动：**
- 在"状态"列后新增"发布状态"列
- 渲染逻辑：
  - 无发布记录：显示空（`-`）
  - 有记录：按平台分行显示，每行：`[平台Icon] [账号名] [发布状态标签]`
  - 发布成功：绿色 Tag + 可点击的链接（`platform_post_url`）
  - 发布失败：红色 Tag + 失败原因 tooltip
  - 发布中/等待中：蓝色/处理中 Tag

**平台 Icon 映射：**
- 新建 `desktop/src/utils/platform.ts`，定义平台名 → icon 的映射
- 使用自定义 SVG icon 或 Ant Design icon 近似替代

## 功能 3：平台管理优化

### 3.1 合并抖音视频和抖音图文

**选定方案**：仅前端兼容显示，后端保持兼容旧名称。

**后端改动 — `server/app/plugins/douyin.py`**
- 保留 `DouyinArticlePlugin` 和 `DouyinVideoPlugin` 不删除
- 新增统一的 `DouyinPlugin` 类：
  ```python
  class DouyinPlugin(BasePublishPlugin):
      platform_name = "douyin"
      display_name = "抖音"
      supported_types = ["article", "video"]
      auth_method = "cookie"
  ```

**后端改动 — `server/app/main.py`**
- 在原有注册基础上，额外注册 `DouyinPlugin()`
- 三个插件并存，旧数据不受影响

**前端改动 — `desktop/src/pages/PlatformManager.tsx`**
- 更新 `platformNameMap`：将 `douyin_article`、`douyin_video`、`douyin` 统一映射为 `'抖音'`
- 平台选择下拉中只显示 `douyin`（统一名称），不显示旧的 `douyin_article`/`douyin_video`
- 已绑定的旧名称账号正常显示为"抖音"

**前端改动 — `desktop/src/utils/platform.ts`**
- 平台 icon 映射中 `douyin_article`、`douyin_video`、`douyin` 均使用抖音 icon

### 3.2 平台选择中增加 icon 显示

**新建文件：`desktop/src/components/PlatformIcon.tsx`**
- 接受 `platformName` prop，返回对应的平台 icon 组件
- 为每个平台准备一个简单的 SVG icon 或使用 Ant Design icon 近似：
  - `wechat_mp` → 微信 icon（自定义 SVG）
  - `wechat_channels` → 微信视频号 icon
  - `xiaohongshu` → 小红书 icon
  - `bilibili` → Bilibili icon
  - `douyin` → 抖音 icon
- 各平台使用品牌色

**修改 `desktop/src/pages/PlatformManager.tsx`**
- 平台卡片标题中增加 `PlatformIcon` 组件
- 绑定平台下拉选项中增加 icon 显示

**修改 `desktop/src/components/PublishModal.tsx`**（功能 2.4 中新建）
- 账号列表中每个平台前显示 `PlatformIcon`

**修改 `desktop/src/pages/ArticleList.tsx`**（功能 2.5 中修改）
- 发布状态列中使用 `PlatformIcon` 显示平台 icon

### 3.3 微信公众号 API 配置和发布

**后端改动 — `server/app/plugins/wechat_mp.py`**

**完整实现微信公众号 API 集成**（草稿 + 发布）：
- `auth_method` 保持 `"oauth"`
- 配置字段存储在 `PlatformAccount.config` JSON 中：
  - `app_id`: 公众号 AppID
  - `app_secret`: 公众号 AppSecret
- `test_connection()`：调用 `https://api.weixin.qq.com/cgi-bin/token` 获取 access_token 验证配置
- `publish()` 完整流程：
  1. 获取 access_token（`grant_type=client_credential`）
  2. 上传图文内的图片素材（获取 thumb_media_id）
  3. 创建草稿（`draft/add`）
  4. 提交发布（`freepublish/submit`）
  5. 查询发布状态（`freepublish/get`）
- 返回 `PublishResult` 包含 `platform_post_id`（media_id）和 `platform_post_url`（文章链接）

**后端改动 — `server/app/routes/platforms.py`**
- 新增 `POST /api/platforms/{account_id}/wechat-mp/config` 端点
- 用于配置微信公众号的 `app_id` 和 `app_secret`
- 将配置保存到 `PlatformAccount.config` 字段

**前端改动 — `desktop/src/pages/PlatformManager.tsx`**
- 绑定微信公众号时，显示额外的配置表单：
  - AppID 输入框
  - AppSecret 输入框（密码类型）
- 已绑定的微信公众号账号卡片中增加"配置"按钮，可修改 AppID/AppSecret
- "测试"按钮调用 test_connection 验证配置是否正确
