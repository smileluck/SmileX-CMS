# 修复本地文件访问 401 Unauthorized

## 问题

前端通过 `<img src="http://127.0.0.1:8000/uploads/xxx.png">` 引用图片时，浏览器发起的 GET 请求不带 `Authorization` header，后端 `/uploads/` 和 `/storage-files/` 端点强制 JWT 认证，导致 401。

## 方案：Token Query 参数

前端在文件 URL 上附加 `?token=xxx`，后端同时支持从 `Authorization` header 和 `token` query parameter 读取 JWT 进行验证。

## 变更清单

### 1. `server/app/dependencies.py` — 新增 `get_current_user_optional` 依赖

在现有 `get_current_user` 旁新增一个支持 query param token 的可选认证函数：

```python
from fastapi import Query

async def get_current_user_for_files(
    request: Request,
    db: Session = Depends(get_db),
) -> User:
    # 1. 先尝试 Authorization header
    auth_header = request.headers.get("authorization")
    token = None
    if auth_header and auth_header.lower().startswith("bearer "):
        token = auth_header[7:]
    
    # 2. 再尝试 query parameter
    if not token:
        token = request.query_params.get("token")
    
    if not token:
        raise HTTPException(status_code=401, detail="Not authenticated")
    
    payload = decode_access_token(token)
    # ... 与现有 get_current_user 逻辑相同
```

### 2. `server/app/main.py` — 文件服务路由改用新依赖

将 `serve_uploaded_file` 和 `serve_storage_file` 的 `Depends(get_current_user)` 改为 `Depends(get_current_user_for_files)`。

### 3. `desktop/src/services/api.ts` — `getMediaUrl` 附加 token

```typescript
getMediaUrl(filePath: string): string {
    const base = this.baseURL.replace('/api', '');
    const token = localStorage.getItem('token');
    const sep = (url: string) => url.includes('?') ? '&' : '?';
    const withToken = (url: string) => token ? `${url}${sep(url)}token=${token}` : url;
    
    if (filePath.startsWith('http')) return filePath;
    if (filePath.startsWith('uploads/')) {
        return withToken(`${base}/${filePath}`);
    }
    // ... 其余逻辑同样包装 withToken
}
```

### 4. `desktop/src/utils/markdown.ts` — `resolveMediaUrl` 附加 token

```typescript
function resolveMediaUrl(url: string): string {
    // ... 现有逻辑生成 resolved URL
    const token = localStorage.getItem('token');
    return token ? `${resolved}?token=${token}` : resolved;
}
```

## 安全性说明

- 本地应用场景，token 通过 URL 参数传递可接受
- URL 中的 token 会出现在浏览器/服务器日志中，但本地应用不涉及跨域传输
- token 有效期 60 分钟，过期后前端会自动跳转登录页
