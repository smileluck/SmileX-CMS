# Plan: 媒体库显示关联文章

## 需求

媒体库页面中，每个媒体文件需要显示其关联的文章标题，让用户知道该媒体被哪篇文章引用。

## 改动概览

需要修改 4 个文件：后端 schema 加字段、后端路由 eager load 文章、前端类型定义加字段、前端页面显示。

---

### 1. `server/app/schemas/media.py` — MediaResponse 增加 article_title 字段

```python
class MediaResponse(BaseModel):
    ...现有字段...
    article_title: Optional[str] = None  # 新增

    class Config:
        from_attributes = True
```

Pydantic 的 `from_attributes = True` 会自动从 ORM 对象的属性读取。由于 SQL 查询会 eagerly load `media.article`，可以在响应构建时从 `media.article.title` 取值。但 Pydantic 不会自动计算嵌套关系属性为 `article_title`，需要在路由层或用 `@computed_field` 处理。

更简单的方案：用 Pydantic 的 `model_validator` 或在路由中手动设置。最简洁的做法是在路由返回前为每条 media 设置一个临时属性。

**最终方案**：在 `MediaResponse` 中加 `article_title` 字段，在路由查询时用 `joinedload` 加载关联 article，然后通过自定义 validator 从 `media.article.title` 取值：

```python
from pydantic import model_validator

class MediaResponse(BaseModel):
    ...现有字段...
    article_title: Optional[str] = None

    @model_validator(mode='before')
    def set_article_title(cls, values):
        if hasattr(values, 'article') and values.article:
            values.article_title = values.article.title
        elif isinstance(values, dict) and 'article' in values:
            values.article_title = values['article']['title']
        return values
```

### 2. `server/app/routes/media.py` — get_media_files 路由 joinedload article

在 `get_media_files` 查询中增加 `joinedload(Media.article)`，确保关联文章数据被加载：

```python
from sqlalchemy.orm import joinedload

@router.get("", response_model=List[MediaResponse])
def get_media_files(...):
    q = db.query(Media).filter(Media.user_id == current_user.id)
    q = q.options(joinedload(Media.article))  # 新增
    if media_type:
        q = q.filter(Media.media_type == media_type)
    if article_id:
        q = q.filter(Media.article_id == article_id)
    return q.order_by(Media.created_at.desc()).offset(skip).limit(limit).all()
```

同时在 `get_media_files` 使用 `joinedload` 后需要 `unique()` 防止重复行：

```python
    return q.order_by(Media.created_at.desc()).offset(skip).limit(limit).unique().all()
```

### 4. `desktop/src/pages/MediaLibrary.tsx` — 卡片 description 显示关联文章

在 `Card.Meta` 的 `description` 中，当 `m.article_title` 存在时显示关联文章标题：

```tsx
<Card.Meta
  title={m.filename}
  description={
    <>
      {m.file_size ? `${(m.file_size / 1024).toFixed(1)} KB` : ''}
      {m.article_title && <div>关联文章：{m.article_title}</div>}
    </>
  }
/>
```
