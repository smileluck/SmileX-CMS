# Plan: 调整保存目录结构（第二轮）

## 背景

上一轮已实现：
- Settings 模型/API（`articles_dir` / `videos_dir` 可配置）
- 目录格式 `{title}-{snow_id}/index.md` + `images/` 子文件夹
- 图片上传 bug 修复 + `images/` 时间戳命名
- `/article-files/` serve 路由

本轮两个需求：
1. 视频类内容与图文类内容存放格式相同，只是存放在 `videos/` 文件夹下
2. `articles/` 和 `videos/` 下插入用户 ID 层级

---

## 变更 1: 视频使用 `videos/` 目录

视频（`article_type == "video"`）和图文使用相同的目录格式（`{title}-{snow_id}/index.md` + `images/`），只是存放在 `videos/` 而非 `articles/` 下。

**修改文件：**

### `server/app/routes/settings.py`
- 不变（已有 `get_videos_dir()`）

### `server/app/routes/articles.py`
- 导入 `get_videos_dir`
- 新增辅助函数：
  ```python
  def _get_content_dir(article_type: str, db: Session, user_id: int) -> Path:
      if article_type == "video":
          return _get_videos_dir(db, user_id)
      return _get_articles_dir(db, user_id)
  ```
- `create_article` / `update_article` / `delete_article` / `duplicate_article` 均使用 `_get_content_dir(article.article_type, db, current_user.id)` 替代原来的 `_get_articles_dir(db, current_user.id)`

### `server/app/routes/media.py`
- 导入 `get_videos_dir`
- `_resolve_article_dir()`: 根据 `article.article_type` 选择基础目录
  ```python
  if article.article_type == "video":
      content_dir = _get_videos_dir(db, user_id)
  else:
      content_dir = _get_articles_dir(db, user_id)
  ```
- `upload_to_article` 返回的 `file_path`：存包含用户 ID 的完整路径（如 `articles/1/{title}-{snow_id}/images/xxx.png`），前端根据前缀选路由

### `server/app/main.py`
- import `VIDEOS_DIR`
- 新增 `GET /video-files/{file_path:path}` serve 路由（与 `/article-files/` 对称）

### `desktop/src/services/api.ts`
- `getMediaUrl()`: 根据 `articles/` 或 `videos/` 前缀，分别使用 `/article-files/` 或 `/video-files/` 路由

## 变更 2: `articles/` 和 `videos/` 下增加用户 ID 层级

### 当前目录结构：
```
articles/
└── {title}-{snow_id}/
    ├── index.md
    └── images/
```

### 目标目录结构：
```
articles/                           ← 可在设置中配置基础路径
└── {user_id}/                      ← 新增：用户 ID 层级
    └── {title}-{snow_id}/
        ├── index.md
        └── images/
            └── 20260406_{snow_id}.png

videos/                             ← 视频独立目录，结构相同
└── {user_id}/
    └── {title}-{snow_id}/
        ├── index.md
        └── images/
            └── 20260406_{snow_id}.png
```

### `server/app/routes/settings.py`

`get_articles_dir()` 和 `get_videos_dir()` 改为返回 `{base}/{user_id}/`：

```python
def get_articles_dir(db: Session, user_id: int) -> Path:
    val = get_setting_value(db, user_id, "articles_dir")
    if val:
        base = Path(val)
    else:
        base = ARTICLES_DIR
    user_dir = base / str(user_id)
    user_dir.mkdir(parents=True, exist_ok=True)
    return user_dir
```

`get_videos_dir()` 同理。

**影响：** 所有调用 `_get_articles_dir(db, user_id)` 的地方自动获得用户子目录路径，无需其他改动。

### `server/app/main.py`

`/article-files/` 和 `/video-files/` serve 路由不需要改动 — 它们在 `ARTICLES_DIR` / `VIDEOS_DIR` 下按路径查找，自然包含 `{user_id}/` 层级。

### `desktop/src/services/api.ts`

`getMediaUrl()` 不需要改动 — 它根据 `articles/` 或 `videos/` 前缀选择路由，路径中自然包含用户 ID。
