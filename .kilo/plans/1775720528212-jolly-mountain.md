# 图文多版本管理实现计划

## 概述

为文章（图文）实现多版本管理：
- **点击编辑时自动新增版本**：用户从列表点击"编辑"时，后端创建一个新版本快照
- **编辑过程中自动保存到当前版本**：auto-save 更新当前编辑版本的内容，不创建新版本
- **版本历史查看与恢复**：可查看历史版本列表，预览某个版本内容，恢复到某个版本

## 1. 数据库层 — 新增 `ArticleVersion` 模型

**文件**: `server/app/models/article_version.py`（新建）

```python
class ArticleVersion(Base):
    __tablename__ = "article_versions"

    id = Column(Integer, primary_key=True, index=True)
    article_id = Column(Integer, ForeignKey("articles.id", ondelete="CASCADE"), nullable=False, index=True)
    version_number = Column(Integer, nullable=False)       # 每篇文章自增
    title = Column(String(255), nullable=False)             # 快照标题
    content = Column(Text, nullable=False, default="")      # 快照内容
    summary = Column(Text)                                  # 快照摘要
    tags = Column(JSON)                                     # 快照标签名数组
    created_at = Column(DateTime(timezone=True), server_default=func.now())

    article = relationship("Article", back_populates="versions")
```

- `(article_id, version_number)` 组成联合唯一索引
- 存储进入编辑时文章的完整快照（title, content, summary, tags）
- `content` 字段在 auto-save 时会持续更新（更新当前版本的 content）

**修改**: `server/app/models/article.py` — 添加 `versions` relationship:
```python
versions = relationship("ArticleVersion", back_populates="article", cascade="all, delete-orphan",
                        order_by="ArticleVersion.version_number.desc()")
```

**修改**: `server/app/models/__init__.py` — 导出 `ArticleVersion`

## 3. 后端 API 路由

**文件**: `server/app/routes/articles.py`（修改）

### 3.1 修改 `_article_to_response` 函数

### 3.2 新增版本相关端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `POST /api/articles/{id}/versions` | POST | **创建新版本** — 快照当前文章内容，返回新版本。点击编辑时调用 |
| `GET /api/articles/{id}/versions` | GET | **获取版本列表** — 返回 `ArticleVersionBrief[]`，按 version_number DESC 排序 |
| `GET /api/articles/{id}/versions/{version_id}` | GET | **获取版本详情** — 返回完整 `ArticleVersionResponse`（含 content） |
| `PUT /api/articles/{id}/versions/{version_id}` | PUT | **更新版本内容** — auto-save 时更新当前版本的 content |
| `POST /api/articles/{id}/versions/{version_id}/restore` | POST | **恢复版本** — 将指定版本的内容写回文章主体 |

#### 3.2.1 创建版本 `POST /api/articles/{id}/versions`
1. 查询文章，校验权限
2. 查询当前最大 version_number，新版本号 = max + 1
3. 快照 article 的 title/content/summary/tags 创建 ArticleVersion 记录
4. 更新 article 的 `current_version` 记录（可以用 article_metadata 存储，或新增字段）
5. 返回 `ArticleVersionResponse`

#### 3.2.2 更新版本 `PUT /api/articles/{id}/versions/{version_id}`
1. 校验权限
2. 更新指定 version 的 content 字段
3. 同时更新 article 主记录的 content（保持一致）
4. 返回更新后的版本

#### 3.2.3 恢复版本 `POST /api/articles/{id}/versions/{version_id}/restore`
1. 查询目标版本
2. 将版本的 title/content/summary/tags 写回 article 主记录
3. 写入 index.md 文件
4. 同步标签
5. 返回更新后的 article

### 3.3 修改现有 `PUT /api/articles/{article_id}` 端点

在 update_article 中：如果有 `version_id` 参数，同时更新对应 version 的 content。

## 6. 前端编辑器改动

**文件**: `desktop/src/pages/ArticleEditor.tsx`（修改）

### 6.1 编辑器初始化 — 创建新版本

当编辑已有文章时（`id` 存在），在加载文章数据后立即调用 `createArticleVersion`：
```typescript
useEffect(() => {
  if (!id) return;
  // 1. 加载文章
  apiService.getArticleById(Number(id)).then(article => { ... });
  // 2. 创建新版本（进入编辑时）
  apiService.createArticleVersion(Number(id)).then(version => {
    setCurrentVersionId(version.id);
  });
}, [id]);
```

### 6.2 Auto-save — 保存到当前版本

修改 `autoSaveFn`，在保存文章的同时更新版本内容：
```typescript
const autoSaveFn = useCallback(async () => {
  if (!title.trim() || !articleId) return;
  const data: any = { title, content, tag_ids: tagIds };
  await dispatch(updateArticle({ id: articleId, data })).unwrap();
  // 同时更新当前版本
  if (currentVersionId) {
    await apiService.updateArticleVersion(articleId, currentVersionId, { content, title });
  }
}, [title, articleId, content, tagIds, currentVersionId, dispatch]);
```
