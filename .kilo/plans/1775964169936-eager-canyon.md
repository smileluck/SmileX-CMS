# 优化图文编辑：插入列表、表格、图片时插入到光标当前位置

## 问题分析

当前所有工具栏操作（列表、表格、图片等）都通过 `handleInsertMarkdown` 函数**追加到内容末尾**，而不是插入到光标当前位置。

核心问题在 `ArticleEditor.tsx:323-331`：

```ts
const handleInsertMarkdown = useCallback((before, after, placeholder, block, description) => {
    setContent(prev => {
      let base = prev;
      if (block && base.length > 0 && !base.endsWith('\n')) base = base + '\n';
      return base + before + (placeholder || '') + (after || '');  // 始终追加到末尾
    }, description);
}, [setContent]);
```

同样，`handleImageUpload`（第 310、315 行）也是追加到末尾：
```ts
setContent(prev => prev + '\n' + imgMd, '插入图片');
```

## 解决方案

### 方案概述

需要同时支持 **Markdown 模式**（textarea）和 **WYSIWYG 模式**（Milkdown）：

1. **Markdown 模式**：读取 textarea 的 `selectionStart` / `selectionEnd`，在光标位置插入文本
2. **WYSIWYG 模式**：通过 Milkdown 的 ProseMirror EditorState 获取选区位置，使用 `editor.action()` 插入内容

---

### 步骤 1：为 Markdown 模式添加 textarea ref 和光标位置追踪

**文件**：`desktop/src/pages/ArticleEditor.tsx`

- 添加 `textareaRef = useRef<HTMLTextAreaElement>(null)`
- 在 textarea 元素上绑定 `ref={textareaRef}`

### 步骤 2：改造 `handleInsertMarkdown` 函数

**文件**：`desktop/src/pages/ArticleEditor.tsx`

将 `handleInsertMarkdown` 改为根据当前 `editMode` 在光标位置插入：

```ts
const handleInsertMarkdown = useCallback((before, after, placeholder, block, description) => {
    if (editMode === 'markdown') {
        // Markdown 模式：使用 textarea 的 selectionStart
        const textarea = textareaRef.current;
        if (textarea) {
            const start = textarea.selectionStart;
            const end = textarea.selectionEnd;
            setContent(prev => {
                const selectedText = prev.substring(start, end);
                const insertText = before + (selectedText || placeholder || '') + (after || '');
                let prefix = prev.substring(0, start);
                const suffix = prev.substring(end);
                // block 类型确保在行首插入
                if (block && prefix.length > 0 && !prefix.endsWith('\n')) {
                    return prefix + '\n' + insertText + suffix;
                }
                return prefix + insertText + suffix;
            }, description);
            // 恢复光标位置（延迟以等待 React 更新）
            requestAnimationFrame(() => {
                if (textareaRef.current) {
                    const newCursorPos = start + before.length + (placeholder?.length || 0);
                    textareaRef.current.selectionStart = start + before.length;
                    textareaRef.current.selectionEnd = newCursorPos;
                    textareaRef.current.focus();
                }
            });
            return;
        }
    }
    // WYSIWYG 模式或 fallback：仍走追加逻辑
    setContent(prev => {
        let base = prev;
        if (block && base.length > 0 && !base.endsWith('\n')) base = base + '\n';
        return base + before + (placeholder || '') + (after || '');
    }, description);
}, [editMode, setContent]);
```

### 步骤 3：改造 `handleImageUpload` 函数

**文件**：`desktop/src/pages/ArticleEditor.tsx`

图片上传完成后，在光标位置而非末尾插入图片 Markdown：

```ts
const handleImageUpload = useCallback(async (file: File) => {
    try {
        let imgMd: string;
        if (articleId) {
            const media = await apiService.uploadToArticle(articleId, file);
            const imgPath = media.markdown_path || `images/${media.file_path.split('/').pop()}`;
            imgMd = `\n![${file.name}](./${imgPath})\n`;
        } else {
            const media = await apiService.uploadFile(file);
            imgMd = `\n![${file.name}](${media.file_path})\n`;
            setPendingImages(prev => [...prev, { mediaId: media.id, originalPath: media.file_path }]);
        }

        if (editMode === 'markdown') {
            const textarea = textareaRef.current;
            if (textarea) {
                const pos = textarea.selectionStart;
                setContent(prev => {
                    return prev.substring(0, pos) + imgMd + prev.substring(pos);
                }, '插入图片');
                requestAnimationFrame(() => {
                    if (textareaRef.current) {
                        textareaRef.current.selectionStart = pos + imgMd.length;
                        textareaRef.current.selectionEnd = pos + imgMd.length;
                        textareaRef.current.focus();
                    }
                });
            } else {
                setContent(prev => prev + imgMd, '插入图片');
            }
        } else {
            setContent(prev => prev + imgMd, '插入图片');
        }
        message.success('图片上传成功');
    } catch {
        message.error('图片上传失败');
    }
}, [articleId, editMode, setContent]);
```

### 步骤 4：保存并恢复 textarea 光标位置

当 `setContent` 触发 React 重渲染时，textarea 会失去光标位置。需要确保在编辑过程中不丢失光标。

在 `handleInsertMarkdown` 中，使用 `requestAnimationFrame` 延迟设置 `selectionStart/End` 即可解决。

### 步骤 5：可选 - WYSIWYG 模式下通过 ProseMirror 插入

当前 WYSIWYG 模式下工具栏按钮并不直接与 Milkdown 交互。有两种处理方式：

- **方案 A（推荐，简单）**：WYSIWYG 模式下保持追加逻辑，因为 Milkdown 通过 `markdownUpdated` 回调同步内容。用户可以在 WYSIWYG 模式下直接编辑，工具栏操作只在 Markdown 模式下支持光标位置插入。
- **方案 B（高级）**：通过 `editorInstanceRef` 获取 Milkdown 实例，使用 ProseMirror 的 `editor.action()` + `insertNode` API 在光标位置插入。这需要更复杂的实现。

**建议先实施方案 A**，因为 WYSIWYG 模式下用户可以直接通过键盘和内置交互来操作列表、表格等，工具栏主要用于 Markdown 模式。

## 风险与注意事项

1. **异步更新问题**：`setContent` 通过 React state 更新，textarea 的 DOM 更新是异步的。使用 `requestAnimationFrame` 确保在 DOM 更新后再设置光标。
2. **块级元素插入**：对于列表、表格等块级元素，需要确保在行首插入（即光标在行首或在上一行末尾时插入新行）。
3. **选中文字替换**：如果用户选中了文字，应该用 `before + selectedText + after` 替换选中内容（而不是用 placeholder）。
4. **焦点恢复**：插入完成后需要让 textarea 重新获得焦点。
