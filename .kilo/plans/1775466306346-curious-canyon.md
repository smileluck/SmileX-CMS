**V1 范围：** Markdown 编辑 + 实时预览 + 公众号样式 + 复制发布

## 实施步骤

### Phase 1：平台样式预览系统（核心）

**目标：** 预览区支持公众号样式渲染，带手机壳 UI。

#### 1.1 创建平台样式主题文件

新建 `desktop/src/styles/platforms/` 目录，每个平台一个 CSS 文件：

- `wechat-mp.css` — 微信公众号样式（绿色引用条、16px 字体、标题颜色 #191919 等）
- `common.css` — 默认通用样式（提取现有预览样式）

每个文件通过 CSS class 前缀隔离（如 `.preview-wechat-mp h1 { ... }`）。

公众号样式关键特征：
- 正文 16px，行高 1.75，字间距 1px
- h1: 22px, #191919, 加粗, 底部 1px 实线
- h2: 20px, #191919, 加粗
- 引用: 左边框 3px #07C160, 背景 #f7f7f7
- 代码: 背景 #f0f0f0, 圆角 3px
- 图片: 圆角 4px, 阴影
- 链接: #576b95

#### 1.2 创建手机壳预览组件

新建 `desktop/src/components/Preview/PhoneFrame.tsx`：

- 外层：手机壳 SVG/CSS 外框（iPhone 样式，圆角、刘海）
- 内层：可滚动内容区，宽度 375px（标准移动端）
- Props: `children`, `platform`(用于样式切换)

#### 1.3 创建平台预览容器组件

新建 `desktop/src/components/Preview/PlatformPreview.tsx`：

- 接收 markdown HTML 内容
- 包裹 PhoneFrame
- 根据 `platform` prop 动态加载对应 CSS class
- 支持主题切换（light/dark）
- 内部内容区可滚动，与编辑区同步滚动

#### 1.4 改造 ArticleEditor 预览区

修改 `desktop/src/pages/ArticleEditor.tsx`：

- 右侧预览区替换为 PlatformPreview 组件
- 顶部增加平台选择器（Radio/Button 组：公众号、通用）
- 选择公众号时加载公众号 CSS 样式

---

### Phase 2：增强工具栏

**目标：** 提供完整的 Markdown 格式化工具栏。

#### 2.1 创建 EditorToolbar 组件

新建 `desktop/src/components/Editor/EditorToolbar.tsx`：

工具栏按钮组：
- **标题：** H1 / H2 / H3 下拉选择
- **格式：** 加粗(B) / 斜体(I) / 删除线(S) / 行内代码
- **插入：** 无序列表 / 有序列表 / 引用 / 分割线
- **高级：** 代码块 / 链接 / 图片（已有） / 表格
- **操作：** 撤销 / 重做

实现方式：通过 Milkdown 的 `editor.ctx` API（commands）来操作编辑器内容。使用 `CallCommand` 来插入对应格式。

#### 2.2 集成到 ArticleEditor

将新工具栏替换现有的简单按钮行。工具栏固定在编辑区顶部。

---

### Phase 3：一键复制富文本

**目标：** 将预览区渲染的 HTML 复制为富文本格式，可直接粘贴到公众号编辑器保持样式。

#### 3.1 创建复制发布功能

在预览区顶部添加"复制富文本"按钮。

实现逻辑：
1. 获取预览区 DOM 容器的 innerHTML
2. 将内联样式写入 HTML 元素（公众号编辑器不支持外部 CSS，需要内联样式）
3. 使用 `navigator.clipboard.write()` 写入 `text/html` ClipboardItem
4. 显示成功/失败提示

#### 3.2 创建样式内联转换工具

新建 `desktop/src/utils/inlineStyles.ts`：

- 输入：带 class 的 HTML + CSS 样式表
- 输出：内联样式的 HTML
- 解析 CSS 规则，匹配 HTML 元素，将 computed style 写入 style 属性
- 移除所有 class 和 id 属性（减少粘贴时的冲突）

---

### Phase 4：自动保存

**目标：** 编辑过程中自动保存草稿，防止数据丢失。

#### 4.1 实现自动保存 Hook

新建 `desktop/src/utils/useAutoSave.ts`：

- 接收保存函数 + 数据 + 间隔时间（默认 30 秒）
- 使用 `useEffect` + `setInterval`
- 仅在内容有变化时保存
- 页面关闭前（`beforeunload`）保存
- 显示保存状态指示器（已保存 / 保存中...）

#### 4.2 集成到 ArticleEditor

在 ArticleEditor 中使用 useAutoSave hook，自动调用 updateArticle。

---

### Phase 5：同步滚动

**目标：** 编辑区与预览区滚动同步。

#### 5.1 实现同步滚动

在 PlatformPreview 组件中：

- 监听编辑区容器的 scroll 事件
- 按比例映射滚动位置到预览区
- 使用 `requestAnimationFrame` 节流
- 支持开关（用户可关闭同步滚动）

---

### Phase 6：快捷键系统

#### 6.1 基础快捷键

在 ArticleEditor 中注册键盘事件监听：

- `Ctrl+B` — 加粗
- `Ctrl+I` — 斜体
- `Ctrl+1~6` — 标题级别
- `Ctrl+S` — 保存
- `Ctrl+K` — 插入链接
- `Ctrl+Shift+K` — 插入代码块

Milkdown 已内置部分快捷键支持（Bold, Italic 等），需确认哪些需额外注册。


---

## 技术决策

1. **CSS 主题切换**：通过在预览容器上切换 class（如 `.preview-wechat-mp`）来实现不同平台样式，轻量无依赖
2. **富文本复制**：使用 `navigator.clipboard.write()` API 写入 `text/html`，兼容 Chrome/Edge
3. **样式内联化**：自行实现轻量 CSS-to-inline 转换，避免引入额外依赖
4. **手机壳 UI**：纯 CSS 实现，不依赖第三方组件
5. **自动保存**：不修改后端，复用现有 `PUT /api/articles/:id` 接口
