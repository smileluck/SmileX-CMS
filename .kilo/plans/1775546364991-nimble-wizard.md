# Plan: Enable Image Upload in Milkdown WYSIWYG Mode

## Problem

The Milkdown WYSIWYG editor does not support image paste/drop upload. The `@milkdown/plugin-upload` package is already installed (`package.json:23`) but not used. The current `onPaste`/`onDrop` handlers on the outer `<div>` (`ArticleEditor.tsx:283`) only work in the markdown `<textarea>` mode — they do not fire in WYSIWYG mode because Milkdown's internal ProseMirror editor intercepts clipboard/drag events.

When a user pastes or drops an image in WYSIWYG mode, nothing happens — the image is not uploaded and does not appear in the article's `images/` directory.

## Solution

Integrate `@milkdown/plugin-upload` into the `MilkdownEditor` component with a custom `uploader` that:
1. Uploads images to the article's `images/` directory via the existing API (`/api/media/upload-to-article/{id}`)
2. For new articles (no `articleId` yet), uploads to the global media library (`/api/media/upload`) and tracks as pending
3. Returns ProseMirror `image` nodes with properly resolved URLs
4. Shows an upload-in-progress indicator

## Changes

### File: `desktop/src/pages/ArticleEditor.tsx`

**1. Extend `MilkdownEditor` props** to accept `articleId` and `onPendingImage` callback:
```ts
const MilkdownEditor: React.FC<{
  value: string;
  onChange: (val: string) => void;
  editorContainerRef?: React.RefObject<HTMLDivElement | null>;
  articleId: number | null;
  onPendingImage?: (mediaId: number, originalPath: string) => void;
}>
```

**2. Inside the editor setup**, add `@milkdown/plugin-upload` with a custom uploader:
- Import `upload`, `uploadConfig` from `@milkdown/plugin-upload`
- Configure `uploadConfig` with a custom `uploader` function that:
  - Filters for image files
  - For each image: calls `apiService.uploadToArticle(articleId, file)` if `articleId` exists, or `apiService.uploadFile(file)` if not
  - For new articles: calls `onPendingImage` to track the pending image
  - Constructs the image URL via `apiService.getMediaUrl()`
  - Creates ProseMirror `image` nodes using `schema.nodes.image.createAndFill()`
- Add `.use(uploadConfig)` and `.use(uploadPlugin)` to the editor chain

**3. Update `ArticleEditor` parent component**:
- Pass `articleId` and `onPendingImage` props to `MilkdownEditor`
- The existing outer `onPaste`/`onDrop` handlers remain as fallback for the markdown textarea mode

### Key Implementation Details

- The custom uploader must return `Promise<Fragment | Node | Node[]>` — it should create `image` schema nodes with `src` set to the full resolved URL (with auth token)
- The upload widget factory should show a localized message like "上传中..." instead of "Upload in progress..."
- `enableHtmlFileUploader` should be `true` to handle images copied from web pages
- Error handling: failed uploads should show an `message.error()` toast
